# Operação

O METIS pode ser aplicado de formas diferentes, dependendo do nível de organização da equipe.

O framework foi projetado para funcionar tanto em ambientes estruturados quanto em cenários de alto caos operacional.

---

## Direto

Equipes que já possuem um fluxo minimamente funcional podem adotar o METIS diretamente através do AFS.

Exemplos:

- equipes utilizando Scrum ou Kanban
- times com backlog organizado
- existência de algum nível de priorização

Nesses casos:

- o AFS assume o fluxo operacional
- o PRP não é necessário inicialmente

---

## Intervenção

Equipes sem organização ou com alto nível de desordem devem iniciar pelo PRP.

Exemplos:

- demandas chegando de múltiplas fontes
- ausência de priorização
- mudanças constantes de escopo
- interrupções frequentes

Nesses cenários:

- o PRP atua como camada de estabilização
- organiza entrada, prioridade e fluxo
- reduz o caos operacional

Após estabilização, a equipe pode migrar para o AFS.

---

## Transição

O PRP não substitui o AFS.

Ele é utilizado como uma etapa anterior quando necessário.

Fluxo de transição:

`PRP -> estabilização -> AFS`

A partir desse ponto, o AFS passa a ser o modelo principal de operação.

---

## Papel

Dentro do METIS:

- o AFS é o modelo padrão de trabalho
- o PRP é um protocolo de intervenção

Essa separação permite que o framework seja aplicado em diferentes contextos sem perda de eficiência.

---

## Objetivo

A estrutura de adoção busca garantir que:

- equipes organizadas evoluam sem aumentar burocracia
- equipes desorganizadas consigam recuperar controle
- o framework se adapte ao contexto real do time

O METIS não exige um ponto de partida único. Ele se adapta ao nível de maturidade da equipe.
